# Operating Model — the observed task loop

Every substantial round in this repo follows the same eight-step loop. Each
step below cites the commit(s) or doc(s) where the behavior is directly
observable. This is a distillation of what was DONE, not aspiration.

## Step 1 — Restate the task and identify the current phase

Before executing, the round is restated against `docs/development_plan.md`
phase/stage numbering (Phase 0-6, Stages 0-6, retained from spec v0.5.1 "for
traceability"). Every commit subject names its milestone explicitly:
`M1a` (046d0e3), `Milestone 1b` (bd32614), `M2a / Phase 1` (510b79f),
`M2b / Phase 2` (91b982b). Phase statuses are updated honestly in the same
round, including re-marking Phase 5 from "not started" to PARTIAL because the
Claude Code exporter shipped de facto (510b79f: "three recorded drift fixes").

## Step 2 — Check allowed / forbidden scope BEFORE execution

Each round carries hard scope constraints declared up front:

- N=2 / N=3 rounds: `schemas/` FROZEN — friction recorded, never edited in
  (`docs/n2_gate_report.md`; bd32614 verified "schemas/ byte-identical to
  1c4a77f").
- M2a research: read-only — "zero installs, zero executions, zero signups
  (spec 9.5)" (510b79f).
- Concurrent work: 490470d excludes the in-flight M2a files from its commit
  ("disjoint files ... excluded from this commit").

## Step 3 — Architect pins IDs and disciplines in work orders BEFORE execution

Rounds are executed by 3-9 subagents under pinned work orders (every commit
carries `Executed-By: N workflow subagents (wf_...)`). Pinning is visible in
outcomes: pinned step-id discipline caught a misspelled workflow step that the
mechanical check would have silently resolved (`n2_gate_report.md` F13);
Category-7 pass criteria, keyword families, and leakage-probe wording were
fixed BEFORE any model call (5d96fd6).

## Step 4 — Read the governing files for the route

The standard's own change discipline requires reading `CHANGELOG_STANDARD.md`
(every schema change maps to a friction ID), `docs/design_review_2026-07-01.md`
(ratified decisions B1-B12), and the phase gate reports before touching
governed surfaces. Observed: v0.6 and v0.7 revisions each open with the exact
friction items they resolve.

## Step 5 — Minimal viable execution, deterministic first

- Tier-0 deterministic before any model: generators are "deterministic-first
  (tier-0: no LLM, no network)" (91b982b); `docs/model_routing_policy.md` §2.
- Drafts, never governed files: `phc new` scaffolds emit `TODO_FILL` markers
  that FAIL `phc validate` by design — "nothing fabricated, ever" (9532648);
  generators write `.generated.yaml` siblings and refuse overwrite without
  `--force` (`CHANGELOG_STANDARD.md` v0.7.1).
- Uncertainty emits a stub, never a guess: "uncertainty always emits TODO_FILL
  stubs, never guesses" (91b982b).

## Step 6 — Self-check against the rubric before review

The round's own numbers are computed, not asserted: computed scorecards are
"regenerated by the runner only" (f8740f8); gate metrics reproduce via
`python scripts/generator_gate.py` (91b982b); suite counts are stated in every
commit body (98 → 114 → 299 → 335 → 369 → 392 → 555 → 629 → 638 → 730) and
README status lines are synced by a parity agent that itself "caught two stale
299 status lines" (e7f0fc6).

## Step 7 — Layered review, staged commit, CI watch

See `./review_protocol.md`. Every commit body records the reviewers, their
verdicts, what they caught, and the code-review-skill triggers that fired
(`Co-Reviewed-By:` trailers on all 17 commits). Staging is explicit-path with
a file-count assertion (510b79f: "Explicit-path staging (6 files asserted)").

## Step 8 — Update memory and decision records

- Friction → `docs/n2_gate_report.md` (append, never rewrite; "The friction
  text above is historical and unchanged").
- Ratified changes → `docs/CHANGELOG_STANDARD.md` with friction-ID mapping.
- Phase status → `docs/development_plan.md` in the same commit.
- Corrections → correction NOTICES prepended to the affected report, results
  recomputed by the grader; the wrong version's existence is disclosed, not
  deleted (2917804; `docs/comparisons/retracted_paper_outcome_pilot.md`).

## The loop in one line

Restate → phase check → scope check → read route files → pin plan (IDs,
gates, forbidden edits) → minimal deterministic execution with TODO_FILL on
uncertainty → self-check via runners and suite → layered review + explicit
staging + CI → append friction/decisions/corrections to the record.
