# Phase 6 — Evaluation runner (REAL: computed scorecards, hardened graders, pre-registered outcome pilot)

**Status: REAL** — built incrementally across commits `046d0e3` (runner + real transcripts), `f8740f8` (grader registry), `9532648` (grader v2 + external audit), `5d96fd6`/`2917804` (Category-7 pre-registration + pilot). Doctrine: `docs/evaluation_framework.md`.

## Goal

Make every quality claim COMPUTED, never asserted: a runner executes real checks and emits the scorecard; behavioral claims rest on real captured transcripts graded by deterministic, documented graders; anything unmeasured is reported UNSCORED with a reason — never guessed, never silently passed. "Build harness = build agent + build evaluation."

## Allowed work

- **Static layer**: completeness, schema validation, cross-reference integrity, policy bans, verified-ratios, pytest-as-reproducibility-signal — all deterministic.
- **Transcript layer**: real, verbatim, single-turn captured model outputs with provenance frontmatter, both arms (baseline vs system under test). Absolute rule: "a test that cannot be run gets no transcript and is reported UNSCORED, never invented."
- **Grader registry**: per-test `grader:` keys dispatching documented deterministic grader types; unknown key or missing transcript ⇒ UNSCORED with reason.
- **Grader hardening with published divergences**: when a deterministic grader disagrees with a human read, publish BOTH directions in the comparison doc, then fix the grader in a separate change. Real case: `scope_refusal` v1 punished a genuine confined refusal for using the package's mandated section format; v2 separates STRUCTURE from SUBSTANCE (pass = refusal marker AND zero verdict language, regardless of section count). Marker lists are extended under a documented contract ("extend, never loosen ad hoc") with mutation regression tests pinned to the exact transcript sentence.
- **Two-scorecard rule**: self-assessed claims and computed measurements coexist; computed wins on conflict; the computed branch may contain ONLY measured dimensions (schema-enforced) and lists every unmeasured dimension in major_failures/next_actions.
- **Outcome evals (Category 7)** under the five requirements: pre-registration of rules before data; information-cutoff with ACTIVE leakage probes (the model may already know the outcome — probe, don't assume); long-enough windows; benchmark adjustment against a baseline arm; published verbatim transcripts. Missing any one ⇒ the outcome dimension is UNSCORED.

## Forbidden work

- Hand-editing a computed scorecard — regenerated by the runner only, every time.
- Loosening a grader in the same change that benefits from the loosening (mrh_t01: the harness-arm FAIL was published as-is; "grader deliberately NOT loosened in the change that would benefit from it"). Grader fixes land in a SEPARATE commit/PR from the transcripts or data they affect (`evaluation_framework.md` §7.4).
- Fabricating, editing, or trimming transcripts; discarding defective runs (the web-search-enabled leakage run was preserved verbatim alongside the operative tools-disabled re-run).
- Treating a track record or small-n result as a capability claim: the n=3 pilot is "proof-of-instrument, not a capability benchmark — no accuracy claim"; the report itself warns where a split is partly a keyword-coverage artifact.
- Silent passes of any kind: skipped person-checks on person-less packages are RECORDED as not_applicable, never dropped.

## Required outputs

1. Runner emitting `scorecard_computed.yaml` (`scoring_basis: computed`) with per-check `computed_detail`.
2. Transcript directories with provenance frontmatter + a no-fabrication README rule, test-enforced.
3. Narrated comparison docs publishing FAILs and grader-vs-human divergences (both arms, both directions).
4. For outcome evals: a pre-registration artifact (pass-criteria keyword families, min-to-pass, leakage-probe wording, contamination markers, exclusion log of rejected candidates) fixed BEFORE any model call — and, per the recorded protocol lesson, **the pre-registration commit must land before any run so git history evidences the ordering** (the pilot's in-session ordering was asserted, not commit-evidenced; the file header discloses this).
5. Post-run audit corrections published, not concealed: rp04's "genuine recognition" contamination claim was found FALSE (grader question-echo artifact) and corrected to SURVIVED-but-UNSCORED with a top-of-file correction notice.

## Acceptance criteria

- Every score in the scorecard traces to an executed check; every unmeasured dimension is explicitly UNSCORED/major_failures.
- Baseline arm run under identical conditions (same excerpts, same tools-disabled condition) — comparisons are benchmark-adjusted, never absolute.
- FAILs of your own system are published and wired into major_failures (M1a: 2 grader false-positives disclosed; demo2: harness FAIL/PASS/FAIL published honestly).
- Grader changes carry regression fixtures from REAL frozen transcripts; regrades reproduce identical verdicts except where the fix is documented.
- Adversarial review of the eval round itself (pre-registration honesty reviewer forced 1 P0 + 3 P1 fixed "by public correction, not concealment").

## Common failure modes

- **Grader measures format, not substance** (scope_refusal v1) — detected only by publishing grader-vs-human disagreement.
- **Retroactive rule-fitting through the back door of "we only fixed the grader"** — prevented by the separate-commit rule.
- **Leakage probe environment underspecified** (N3-F12): the probe WORDING was pre-registered but not the run environment; default config had web search ON, defeating a memory-contamination probe. Pre-register the environment (all tools disabled) and which reading governs when probes disagree.
- **Test asserting a pre-run snapshot forever** (N3-F10): a lifecycle transition (excerpt_verified false→true) is not grader-fitting; document it as such when a test must change.
- **UNSCORED quietly becoming a pass** — UNSCORED must be counted, listed, and carried into next_actions.

## Self-check checklist

- [ ] Is every published number emitted by a script someone else can re-run?
- [ ] Are all transcripts real, verbatim, provenance-stamped — and absent ones UNSCORED?
- [ ] Did any grader change land in the same change as data it benefits? (Must be NO.)
- [ ] For outcome evals: pre-registration committed BEFORE runs? Leakage actively probed? Baseline arm identical? Transcripts published?
- [ ] Are my own FAILs and divergences published and wired into major_failures?
- [ ] Did an adversarial reviewer audit the round's honesty?
